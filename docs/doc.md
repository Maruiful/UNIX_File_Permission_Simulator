# 要求

访问控制技术：建立基于访问权限表的系统安全管理。
⚫ 实现 UNIX/LINUX 的文件访问权限模型；
⚫ 模拟文件及目录的数据结构，为文件添加访问权限
⚫ 建立用户与用户组模型
⚫ 设计读、写和执行三种指令，模拟不同用户访问文件或目录时所发生的受限情
况。





限制：

### 1. 编程语言与开发环境

- 

  **编程语言**：原则上仅允许使用 **C语言**或**汇编语言** 。主要内核代码应使用C语言，不建议使用C++ 。

  

  

- 

  **严禁语言**：不允许使用Java、Visual Basic等解释执行的语言 。

  

  

- 

  **开发环境**：建议在Windows环境下使用 **Microsoft Visual Studio 2019** 或更新版本 。若熟悉Windows编程可使用MFC，否则建议使用Win32 Console Application（控制台应用） 。

  

  

------

### 2. 操作系统代码（内核部分）规范

内核代码完成设计的主体内容，必须遵循以下严格限制：

- 

  **输出物形式**：该部分通常组织为一个Project，目标是生成一个 **.lib（库文件）**，而非直接生成应用程序 。

  

  

- 

  **禁止标准库/语句**：严禁使用标准STL，不允许出现输入输出语句（如 `cin`/`cout`, `printf`/`scanf`）以及任何窗口操作 。

  

  

- 

  **无界面要求**：内核代码必须没有UI，不带任何界面，且没有针对标准I/O的操作 。

  

  

- 

  **设计核心**：强调紧凑、节省空间的数据结构设计，算法应以教材标准算法为基准，鼓励在有理论证明前提下的创新 。

  

  

------

### 3. 演示代码规范

演示代码用于验证内核代码的有效性：

- 

  **输出物形式**：链接结果为一个 **应用程序（如Windows下的 .exe）** 。

  

  

- 

  **功能实现**：需有机地按顺序调用内核函数，展示较大模块的功能和运行过程 。

  

  

- 

  **界面要求**：通常带有UI，可以使用标准窗口或字符命令窗口，要求功能展示完整且操作简便
